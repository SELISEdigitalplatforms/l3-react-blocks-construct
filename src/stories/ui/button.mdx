## Variants

<Subtitle>States & Behavior</Subtitle>

The Button should clearly reflect different states:

- **Default** — ready to click.
- **Hover** — highlight or elevation to show interactivity.
- **Active / Pressed** — visual feedback that the click is being registered.
- **Disabled** — muted appearance, no click event.
- **Loading** — spinner or indicator + disable to prevent duplicate clicks.

## Best Practices

- Use action-oriented, concise text (verbs like **Submit**, **Confirm**, **Cancel**).
- Only one **primary button** per view or flow to avoid confusing calls-to-action.
- Provide sufficient padding and touch target size for click / tap.
- Maintain **high contrast** between text and background for readability.
- Ensure **focus styles** are visible for keyboard navigation.

---

## Accessibility

- Use proper ARIA attributes where needed (`aria-label`, `aria-disabled`, etc.).
- Buttons should be **reachable via keyboard** (Tab / Shift+Tab).
- Active/focus state should be **clearly visible**.
- For **icon-only buttons**, always provide a textual label for screen readers.

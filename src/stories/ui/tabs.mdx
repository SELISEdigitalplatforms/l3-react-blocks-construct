<Subtitle>Overview</Subtitle>

Tabs organize related content into separate views, showing only one section at a time. Commonly used for grouping forms, settings, or content categories.

## Variants

- **Default:** Standard tabs with triggers and content areas.

## Props

- **defaultValue:** Initial tab selection for uncontrolled usage.

- **value:** Controlled active tab value.

- **onValueChange:** Callback when the active tab changes.

- **className:** Custom CSS classes.

- **TabsTrigger:**

  - **value:** Identifier for the tab.

  - **disabled:** Disables the tab.

# Best Practices

- Keep labels short and clear.

- Use tabs only for related but distinct content.

- Ensure accessibility with keyboard navigation.

- Preserve input state when switching tabs.
